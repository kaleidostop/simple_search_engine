# Простая поисковая система

Консольное приложение для полнотекстового поиска по коллекции текстовых файлов с использованием инвертированного индекса и ранжирования BM25.

## Возможности

- **Индексатор** – рекурсивно обходит директорию, строит инвертированный индекс и сохраняет его в файл.
- **Поисковая программа** – загружает индекс и в интерактивном режиме ищет документы по запросам.
- Поддержка логических операторов `AND`, `OR` и скобок.
- Регистронезависимый поиск (слова приводятся к нижнему регистру, знаки препинания заменяются на пробелы).
- Ранжирование результатов по формуле **BM25** (на основе TF-IDF с поправкой на длину документа).
- Вывод списка файлов с указанием номеров строк, где встречаются искомые слова.
- Компактный словарь на основе префиксного дерева.

## Сборка

Требуется компилятор с поддержкой C++20 и CMake (минимум 3.17).

```bash
git clone https://github.com/kaleidostop/my-little-google.git
cd my-little-google
mkdir build
cd build
cmake ..
cmake --build .
```

После сборки исполняемые файлы `indexer` и `search` появятся в папке `bin/` в корне репозитория.

## Использование
Команды выполняются из корня репозитория.
### 1. Индексация
```bash
./bin/indexer ./test_dir
```
- Путь может быть абсолютным или относительным.
- Имена файлов не должны содержать пробелов.
- Индекс сохраняется в файл `inverted_index.txt` в корне репозитория

### 2. Поиск
```bash
./bin/search 
```
Программа загружает индекс и переходит в интерактивный режим. Введите запрос и нажмите Enter. Для выхода введите `q`.

Результат – список файлов и строк, где встречаются искомые слова, отсортированный по релевантности.

## Синтаксис запросов
- Операторы `AND` и `OR` – строго в верхнем регистре, остальные слова регистронезависимы.
- Скобки для группировки.
- Несколько пробелов между словами игнорируются.

Примеры запросов:
- `cat`
- `cat AND dog`
- `(dog OR lemon) AND cat`

Некорректные запросы (например, `cat AND` или `cat dog`) выводят сообщение об ошибке.

## Детали реализации
### Инвертированный индекс
- Словарь термов построен на префиксном дереве (trie):   Каждый узел дерева соответствует префиксу слова и содержит ссылки на дочерние узлы (по одному на каждый возможный следующий символ).  
В конечном узле, соответствующем слову, хранится постинг-лист — список документов, в которых это слово встретилось.
- Постинг-лист для одного терма — это перечень записей для одного терма. Каждая запись содержит:
  - идентификатор документа (`docID`);
  - общее количество вхождений терма в документ (`count`);
  - номера строк, где терм появляется (`lines`).
- Документы идентифицируются относительными путями (с прямыми слэшами). Если один и тот же файл передаётся на индексацию повторно, он игнорируется.

### Препроцессинг текста
- Знаки препинания заменяются на пробелы перед токенизацией
- Все слова приводятся к нижнему регистру 

### Сохранение и загрузка индекса
Индекс сохраняется в текстовый файл рекурсивным обходом дерева с сохранением метаданных.

Сохраняются: общее количество документов, суммарная длина всех документов, имена файлов, длина каждого документа, а затем структура trie с постинг-листами.  
При загрузке дерево полностью восстанавливается.

### Поиск и ранжирование
- Запрос разбирается в обратной польской нотации с помощью алгоритма сортировочной станции (стек операторов и операндов).
- Логические операции AND/OR обрабатываются над множествами документов:
    - AND оставляет документы, содержащие все термы.
    - OR — оставляет документы, содержащие хотя бы один терм.
- Ранжирование выполняется по формуле BM25.  
Для каждого документа, прошедшего логический фильтр, вычисляется сумма BM25 по всем термам запроса. 
  
  $$BM25(t, d) = \frac{tf(t,d) \cdot (k+1)}{tf(t,d) + k\cdot\left(1 - b + b \cdot \dfrac{dl(d)}{dl_{avg}}\right)} \cdot ln\left(\dfrac{N - df(t) + 0.5}{df(t) + 0.5}\right)$$

    - $TF$ (частота терма в документе) показывает, сколько раз слово встретилось в данном документе
    - $DF$ (документная частота) – в скольких документах коллекции это слово встречается вообще.  
    - $N$ – число документов, $dl$ – длина документа
    - Параметры $BM25: k = 1.2, b = 0.75$.

- Номера строк в выдаче уникальны и отсортированы по возрастанию.

## Возможные улучшения
- Бинарный формат индекса – вместо текстового для уменьшения размера и ускорения загрузки.
- Сжатие постинг-листов (Variable Byte Encoding, скип-листы) для работы с большими коллекциями.
- Фразовый поиск – использование точных позиций слов.
- Стемминг или лемматизация для повышения качества поиска.
- Хранение индекса на диске (без полной загрузки в память) для масштабирования.
- Потокобезопасный параллельный поиск.
